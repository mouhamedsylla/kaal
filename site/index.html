    pilot : Documentation

    Dev Environment as Code : de l'init au déploiement, sans friction.

    Guides pratiques

    Commence ici si tu découvres pilot.

    • Démarrer avec pilot : wizard d'init, génération des fichiers d'infra, premier pilot up
    • Ajouter un service : pilot add pour étendre l'infra après l'init (Redis, Neon, Cloudflare R2…)
    • Premier déploiement sur VPS : preflight, push, deploy, rollback

    Comprendre pilot

    • Concepts & Philosophie : Pourquoi pilot existe, le modèle mental, la valeur ajoutée
    • Architecture du code : Packages, interfaces, hexagonal architecture, pipeline de déploiement
    • Modèle de résilience : Taxonomie TypeA/B/C/D, machine à états, pilot.lock

    Référence

    • pilot.yaml : Référence complète : Tous les champs : project, registry, services (hosting/provider/catalogue), environments, targets
    • Workflows complets
    • Toutes les commandes
    • Outils MCP : Liste des outils et protocole JSON-RPC

    Workflows complets

    • Développement local : De pilot init à pilot up
    • Intégration Agent IA : Générer l'infra et déployer avec Claude/Cursor
    • Déploiement VPS : Preflight, push, plan, deploy, rollback
    • CI/CD : pilot dans GitHub Actions / GitLab CI

    Commandes

    Infra locale

    Commande Description
    pilot init Initialiser pilot dans un projet nouveau ou existant
    pilot add Ajouter un service à un projet existant
    pilot up / pilot down Démarrer / arrêter l'environnement local
    pilot env Gérer les environnements (use, current, diff)

    Déploiement

    Commande Description
    pilot push Build + push de l'image Docker vers le registry
    pilot deploy Déployer sur une cible distante (pipeline 8 étapes)
    pilot rollback Revenir à la version précédente
    pilot sync Synchroniser les fichiers de config vers le VPS
    pilot preflight Vérifier les prérequis + générer pilot.lock
    pilot plan Afficher le plan d'exécution sans déployer
    pilot setup Préparer un VPS vierge pour le déploiement

    Observation

    Commande Description
    pilot status État complet du projet et des services
    pilot logs Logs d'un service (local ou distant)
    pilot diagnose Snapshot complet du système
    pilot history Historique des déploiements

    Secrets

    Commande Description
    pilot secrets Gérer les secrets (list, get, set, inject)

    Agent IA & MCP

    Commande Description
    pilot mcp serve Démarrer le serveur MCP (JSON-RPC 2.0 stdio)
    pilot mcp context Afficher le contexte complet pour un agent AI

    Divers

    Commande Description
    pilot resume Reprendre une opération suspendue (TypeC)
    pilot update Mettre à jour pilot vers la dernière version

    Troubleshooting

    • Variables d'environnement vides dans le container : ARG/ENV pattern, priorité des vars, fixes intégrés
    • Compose désynchronisé après pilot add : Staleness detection, comment régénérer

    Internals (pour contribuer)

    • Moteur de contexte : internal/mcp/context : collecte du contexte, AgentPrompt, MissingComposeEnvs
    • ExecutionProvider : compose, k8s : runtime local
    • DeployProvider : VPS/SSH (interface, Sync, hooks, migrations, erreurs TypeC/D)
    • Serveur MCP : Protocole JSON-RPC 2.0, outils implémentés
