    pilot : Documentation

    Dev Environment as Code : de l'init au déploiement, sans friction.

    Navigation

    Comprendre pilot

    • Concepts & Philosophie : Pourquoi pilot existe, le modèle mental, la valeur ajoutée
    • Architecture du code : Packages, interfaces, patterns Go
    • pilot.yaml : Référence complète : Schéma, champs, exemples

    Commandes

    • pilot up / down : Démarrer / arrêter l'environnement local
    • pilot context : Exporter le contexte pour un agent AI
    • pilot push : Build + push de l'image Docker (auto-inject VITE_*, platform detection)
    • pilot deploy : Déployer sur une cible distante (sync automatique, rollback auto)
    • pilot sync : Synchroniser les fichiers de config vers le VPS sans redéployer
    • pilot rollback : Revenir à la version précédente
    • pilot status : État complet du projet et des services
    • pilot logs : Consulter les logs d'un service
    • pilot env : Gérer les environnements (switch, liste)
    • pilot setup : Préparer un VPS pour le déploiement
    • pilot history : Historique des déploiements
    • pilot mcp : Démarrer le serveur MCP (JSON-RPC 2.0 stdio)
    • pilot preflight : Vérifier tous les prérequis avant de déployer

    Workflows complets

    • Développement local : De pilot init à pilot up
    • Intégration Agent AI : Générer l'infra et déployer avec Claude/Cursor
    • Déploiement VPS : Preflight, push, deploy, rollback
    • CI/CD : pilot dans GitHub Actions / GitLab CI

    Troubleshooting

    • Variables d'environnement vides dans le container : ARG/ENV pattern, priorité des vars, fixes et protections intégrées

    Internals (pour contribuer)

    • Moteur de contexte : Comment internal/context collecte les informations
    • Orchestrateurs : compose, k3d, k8s
    • Providers : VPS/SSH (interface, Sync, CopyFileTo, bind-mounts)
    • Serveur MCP : Protocole JSON-RPC 2.0, outils implémentés
